Beginning

The idea behind this application was to create a platform where you will be able to book services like salons, accountants, lawyers, hospitals and gyms.

The deliverable expected from the design team were, user website and application, vendor website and application, and admin panel. As the platform will be open for so many services we needed to come up with a design system which would do justice to all the services. We developed our color palatte around the logo of our client.

User Application & Wesbite

After discussing with the client and software team, basic structure of the application decided. According to which we were going to begin with salon services.

For the first time I was building an User interface from scratch and my approach was to get it as minimal and appealing as possible. Sticking to color palatte made high fedility designs of the wireframes and was majorly focusing on making the workflow of booking a service user friendly.

Below we have screens showing the flow for booking a service (salon service), here a unique feature was to select the employee from whom you would like to take the service.

Vendor Application & Wesbite

Vendor application has to be more structured than the user, as every facility visibly on the user website would be editable by the vendor for ther service.

Here vendors will create their profile and then purachase a subscription to list their business/service, then they have to list their services and employees. They will have option to edit all the deatils at any point of time until the subscription expires. All the details are needed to be added by the vendor to list heri business and this can be a hectic task, an alternative suggested by the team was to take information the vendor then list their business. This will also help to retain the identity of the application.

Below we have screens showing the flow for listing a service on the application.

Admin Application

Admin application was having all features for managing both user and vendors as well.

Got to know about admin application while working here and how it functions for every user based application. Admin panel on a whole has to be minimilastic and it provides the administrator to access and change the complete working of the application. Features in the panel were to create and manage subscriptions for vendors and manage all the booking data of users.

Screens representing the included features are presented.

Concluding

This application was a great learning experience, as I got to develop a complete user based application from scratch.

Here we apllied all our UI & UX knowledge to make a successful application and our first deliverable was quite impressive according to the client. Learnt how designing principles vary from all perspectives, in this case for users, vendors and administrator. Admin panel was having a minimal presentation in contrast to the main application as they both were serving different purpose.

Gained knowledge of how to make an application for multiple services and still retaining the identity.
